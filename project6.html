Multicycle RISC Processor Design

A project simulating a multicycle CPU architecture using Verilog

Objective

Design and simulate a RISC processor based on a Multicycle model, separating each stage: Fetch, Decode, Execute, Memory, and Write Back. The goal is to reduce hardware resources compared to pipelined CPUs while ensuring proper control and data flow.

Main Components

  • ALU, Register File, Control Unit, Memory
  • Instruction Memory, Main Memory (RAM)
  • Program Counter, Sign Extend, MUX, ALU Control
  • Language: Verilog HDL
  • Simulated using Vivado 2023.2

CPU Block Diagram

Operation Flow

  1. Instruction Fetch: PC sends address to Instruction Memory, fetches instruction
  2. Instruction Decode: Decode opcode, read two registers from Register File
  3. Execute: ALU performs operation or calculates address
  4. Memory Access: Access RAM if needed
  5. Write Back: Write result back to register
